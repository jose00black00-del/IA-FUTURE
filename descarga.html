Instalación de VALEN-LLAMA

Esta sección describe, de forma resumida, cómo se prepara un equipo convencional para ejecutar el asistente VALEN-LLAMA sin conexión a internet. No es un manual técnico completo, sino una guía clara para reproducir el entorno de pruebas usado en la tesis.

Requisitos mínimos sugeridos

Equipo del laboratorio

  • • Sistema operativo de escritorio (por ejemplo, Windows).
  • • Procesador convencional de 64 bits.
  • • Al menos 8 GB de memoria RAM recomendados.
  • • Espacio libre en disco para el modelo de IA y los archivos del proyecto.
  • • Python instalado y funcionando correctamente.

El objetivo es demostrar que no se necesitan servidores externos: todo ocurre dentro de la misma máquina del laboratorio.

Flujo general

Qué se instala exactamente

La instalación se divide en tres partes:

  • 1. Entorno de Python y dependencias básicas.
  • 2. Archivos del proyecto VALEN-LLAMA (código y recursos).
  • 3. Modelo de lenguaje local y módulo de asociaciones.

Una vez completados estos pasos, el asistente se ejecuta desde una ventana de consola, sin necesidad de internet.

Pasos de instalación

A continuación se muestra una ruta de instalación simplificada, pensada para replicar el prototipo en otros equipos del centro.

1. Preparar el entorno

  • 1.1 Verificar que el sistema operativo esté actualizado y que el usuario tenga permisos para instalar software.
  • 1.2 Instalar Python desde un instalador confiable y activar la opción de añadirlo a la variable de entorno del sistema.
  • 1.3 Crear una carpeta dedicada al proyecto, por ejemplo: C:\VALEN_LLAMA\.
C:\VALEN_LLAMA\   // Carpeta principal del asistente

2. Copiar los archivos del proyecto

  • 2.1 Copiar en la carpeta del proyecto los archivos de código del asistente (scripts en Python, configuraciones y recursos).
  • 2.2 Mantener una estructura sencilla de subcarpetas para distinguir: código, datos del modelo y asociaciones.
  • 2.3 Comprobar que todos los archivos necesarios están presentes antes de continuar.
C:\VALEN_LLAMA\
├─ codigo\
├─ modelo\
└─ asociaciones\

3. Configurar el modelo local

  • 3.1 Copiar el modelo de lenguaje local en la carpeta asignada (por ejemplo, modelo\).
  • 3.2 Ajustar en el código la ruta donde se carga el modelo, de modo que el asistente siempre lo busque en disco y no en la nube.
  • 3.3 Probar un inicio rápido para comprobar que el modelo se carga sin errores de memoria.

4. Ejecutar VALEN-LLAMA

  • 4.1 Abrir una ventana de consola en la carpeta del proyecto.
  • 4.2 Iniciar el asistente mediante el comando configurado (por ejemplo, ejecutar un archivo .py principal).
  • 4.3 Esperar a que aparezca el mensaje de bienvenida y verificar que el sistema responde a una pregunta sencilla.
python codigo\valen_llama_main.py

Lista rápida de verificación

Antes de presentar el sistema en el laboratorio o en la defensa de la tesis, conviene revisar estos puntos.

Prueba sin internet Confirmar que el asistente arranca y responde correctamente con la conexión desactivada.
Consumo de recursos Revisar el uso de CPU y RAM mientras se hacen varias preguntas seguidas para comprobar que el equipo lo soporta.
Escenarios de demostración Preparar algunas preguntas de ejemplo (matemática, física, programación) para mostrar el sistema en acción.
Nota importante: VALEN-LLAMA es un prototipo académico. La instalación descrita aquí está pensada para laboratorios controlados y equipos específicos del centro. Cualquier adaptación a otros entornos debe respetar siempre las políticas de seguridad informática y el uso responsable de los recursos del laboratorio.
